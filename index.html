Obradoiro git

Qué é GIT

  • Git é un sistema de control de versións
  • Serve pra levar control dos contidos dos nosos proxectos
  • https://git-scm.com

Conceptos: Repositorio

Un repositorio é un árbol de carpetas ou directorios que conten os ficheiros co traballo que nos interesa versionar e serán vixiados por GIT

git init: Crear un repositorio inicial

  • O primer paso pra usar git é declarar cal vai ser a ruta raíz do repositorio
  • Basta con situarse no directorio desexado e executar git init
  • Ex: 
    $ git init

git add: Engadir contido ao noso repositorio

  • Pra controlar e versionar os nosos ficheiros, o primeiro é engadilos
  • Con git add comunicamos a GIT que queremos seguemento destes ficheiros
  • Ex: 
    $ git add novoFicheiro.html

git status: Qué carallo pasa aquí, ho

  • git status infórmanos sobre o estado actual do repositorio

Conceptos: Working tree, index

Conceptos: Commit

  • Un commit é un conxunto de cambios agrupados con significado propio
  • Gravamos commits no repositorio que cambian o seu estado
  • Por tanto, gardar ou aplicar un commit, supón cambiar os ficheiros
  • Ao facer cada commit, débese introducir unha mensaxe que describe o cambio que provoca

git commit: Gardando os nosos cambios

  • Con este commando creamos un commit que agrupa todos os cambios engadidos ata o momento
  • Tralo commit, o traballo feito nos ficheiros e o código se considera "un paso" máis avanzado
  • Ex: 
    $ git commit -m "Amplía o tamaño do botón"
  • Como podemos ver, este commit fai un botón máis grande.
